Разборка (разукомплектация изделия)

Этот раздел описывает логику разборки (разукомплектации) изделия, реализованную через производственный заказ специального типа.

Общая идея​

Разборка — это «обратное производство»:

  • исходное изделие списывается из места хранения (как материал);
  • компоненты из спецификации поступают в место хранения готовой продукции (как выпуск).

Фактически система оформляет операцию:

  1. списать исходное изделие;
  2. оприходовать набор компонентов, рассчитанный по спецификации.

Что нужно для работы разборки​

  1. Должен быть создан тип производственного заказа с признаком «Разборка».
  2. Для разборки должна быть задана спецификация (состав), по которой система понимает, на что разбирать изделие.

Примечание: в начальных данных обычно уже есть тип «Разборка», помеченный как разборка.

Чем разборка отличается от обычного производства​

Какие строки считаются «материалами», а какие — «выпуском»​

При обычном производстве:

  • материалы = компоненты из спецификации;
  • выпуск = готовое изделие (и возможные побочные продукты).

При разборке:

  • материалы = само исходное изделие (и дополнительные позиции, если они указаны как побочные продукты в спецификации);
  • выпуск = компоненты спецификации.

Плановое количество​

Поле «К изготовлению» в заказе используется и для разборки, но смысл меняется:

  • при разборке это количество исходного изделия, которое будет разобрано.

Автоматическое заполнение строк по спецификации​

В заказе доступно действие формирования строк (обычно «Создать строки», доступно в статусе «Черновик»).

При разборке система делает следующее:

  1. Очищает текущие строки материалов и выпуска.
  2. По компонентам спецификации создаёт строки выпуска:
    • номенклатура = компонент;
    • количество к производству рассчитывается пропорционально:
      • количество компонента в спецификации × количество разборки / количество спецификации;
    • если у компонента задан коэффициент распределения себестоимости, он переносится в строку выпуска.
  3. Создаёт строку материала для исходного изделия:
    • номенклатура = изделие заказа;
    • количество к списанию = количество разборки.
  4. Если в спецификации указаны побочные продукты, они добавляются как дополнительные строки материалов (то есть тоже к списанию).

Резервирование и запуск разборки​

Разборка проходит по тем же статусам, что и обычное производство:

  1. Подготовьте заказ: тип «Разборка», изделие, спецификация, количество.
  2. Проверьте наличие и зарезервируйте материалы.
    • для разборки материалом является исходное изделие;
    • если исходного изделия нет в месте хранения материалов, заказ не сможет перейти в готовность.
  3. Переведите заказ «в работу».
  4. Зафиксируйте фактическое выполнение:
    • при разборке «произведено» считается по факту списания исходного изделия;
    • система может автоматически пропорционально заполнить выпуск компонентов и списание исходного изделия.
  5. Завершите заказ и укажите место хранения готовой продукции, куда поступят компоненты.

Себестоимость при разборке​

Логика оценки та же, что и для производства:

  • основа себестоимости — фактическое потребление (для разборки это списание исходного изделия и дополнительных строк материалов);
  • итоговая сумма распределяется на строки выпуска (компоненты) по коэффициентам распределения.

Если коэффициенты распределения не заполнены, вся себестоимость может уйти на одну строку выпуска.

Подробно см. Себестоимость: как рассчитывается.

Типовые ошибки​

  • Не формируются строки разборки — не выбрана спецификация.
  • Не удаётся зарезервировать — в месте хранения материалов нет исходного изделия в нужном количестве.
  • Компоненты поступают не в то место хранения — проверьте, что при завершении указано место хранения готовой продукции.
