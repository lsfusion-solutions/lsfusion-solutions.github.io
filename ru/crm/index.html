Пользовательская документация по работе с лидами

Этот комплект документов описывает пользовательскую работу с лидами: ведение карточки, работа со списком, доска по статусам, обработка звонков и писем, создание связанных документов, отчётность и настройка справочников.

С чего начать​

  1. Откройте раздел «Лиды» (в дереве навигации он находится в группе «Операции»).
  2. Создайте лид (кнопка «Новый» в списке).
  3. Заполните минимум, чтобы лид было удобно вести:
    • Наименование
    • Менеджер по продажам
    • Тип лида и Статус лида (если у вас настроена воронка)
    • при наличии — Контрагент, Телефон, Электронная почта
  4. Для контроля используйте:
    • фильтры «Открыт», «Закрыта», «Мои лиды»;
    • вкладку «Канбан» (колонки по статусам, быстрый перенос между этапами).

Подробные инструкции:

  • Лиды: список и карточка
  • Доска по лидам (канбан)
  • Коммуникации: звонки и письма
  • Заказы и реализации из лида
  • Отчет по лидам

Термины и ориентиры​

Лид​

Потенциальная продажа или обращение, которое проходит через этапы работы (статусы). Лид может быть привязан к контрагенту, ответственному сотруднику, приоритету и набору ярлыков.

Тип лида​

Классификация лида. Тип влияет на то, какие статусы допустимы для лида.

Статус лида​

Этап работы с лидом. Статус может быть «закрытым» (такие статусы не показываются на доске по лидам).

Поражение и причина поражения​

Специальный сценарий закрытия лида: пользователь фиксирует поражение и выбирает причину. После этого лид получает статус, назначенный в настройках как «поражение».

Приоритет лида​

Важность лида. Приоритет может иметь цветовую индикацию (например, для быстрого визуального поиска в списке).

Ярлыки лида​

Метки для группировки лидов (например, «выставка», «повторный контакт», «срочно»). Ярлыки тоже могут иметь цвет.

Типовые сценарии​

  1. Поступил входящий звонок / письмо → создайте лид по коммуникации или прикрепите коммуникацию к существующему лиду → далее ведите лид по статусам.
  2. Появилась потребность в документе → из лида создайте заказ или реализацию (если это настроено) → контролируйте связанные документы в карточке лида.
  3. Нужно проанализировать воронку → используйте Отчет по лидам и фильтры по интервалу дат.

Примечания​

  • Доступность отдельных вкладок и действий зависит от прав пользователя и настроек (например, для создания заказа/реализации должны быть заданы соответствующие типы документов).
  • В документации используются названия разделов/полей/кнопок так, как они отображаются в интерфейсе.

Роли и ответственность​

Менеджер по продажам​

Отвечает за:

  • создание и ведение лидов;
  • поддержание актуального статуса;
  • фиксацию договорённостей в «Описание»;
  • закрытие лида как успешного или через «Поражение» с причиной.

Руководитель​

Отвечает за:

  • настройку воронки (статусы, типы, причины поражений);
  • контроль качества заполнения лидов;
  • анализ отчётов и нагрузки по менеджерам.

Ежедневный регламент работы​

  1. Откройте раздел «Лиды» и включите фильтры «Открыт» и «Мои лиды».
  2. Просмотрите лиды без ожидаемой даты закрытия и без заполненного описания — это первые кандидаты на уточнение.
  3. Переключитесь на доску по лидам и проверьте, нет ли лидов, которые «застряли» на одном статусе.
  4. Обработайте вкладки «Звонки» и «Электронная почта» (если они доступны):
    • создайте лиды по новым обращениям;
    • прикрепите коммуникации к существующим лидам;
    • отметьте «Игнорировано», если обращение не относится к продажам.
  5. Зафиксируйте следующий шаг в описании лида (что делать и когда).

Частые вопросы​

Почему я не вижу кнопку создания заказа или реализации?​

Проверьте в разделе «Настройка → Настройки», заданы ли параметры «Тип заказа» и «Тип реализации». Если параметры не заданы или у вас нет прав, кнопки не отображаются.

Почему не удаётся сохранить лид после смены типа?​

Тип лида может ограничивать допустимые статусы. После смены типа выберите статус, который разрешён для этого типа.

Почему не отображается доска по лидам?​

Чаще всего причина в том, что статусы помечены как «Закрыта», либо для выбранного типа лида не разрешены статусы.
