Отчёты по продажам

Раздел отчетности используется для анализа:

  • объёма продаж;
  • выполнения заказов;
  • отгрузок и реализаций;
  • скидок;
  • дебиторской задолженности (если включено).

Отчеты доступны в меню модуля Продажи в разделе Отчетность. В зависимости от прав доступа и включенных функций набор доступных отчетов может отличаться.

Модуль Продажи включает следующие отчеты:

  • Отчет по ценам — анализ и сравнение цен номенклатуры на выбранные даты, с опциональным отображением остатков.
  • Отчет по заказам — подробный список заказов покупателей и их строк с суммами, налогами и атрибутами номенклатуры.
  • Отчет по продажам — детальный отчет по регистру продаж с количеством, выручкой, себестоимостью и маржой.
  • Сводка по продажам — дашборд с графиками трендов продаж по неделям, покупателям, местам хранения и категориям.

Отчет по ценам​

Отчет по ценам позволяет анализировать цены на продукцию на определенные моменты времени, сравнивать их между двумя датами и просматривать остатки на выбранных складах.

Основные возможности​

  • Анализ цен: просмотр цен для всех или выбранных видов цен на определенную Дату (дата и время).
  • Сравнение цен: сравнение цен между двумя датами. Отчет показывает цены на обе даты и процентную разницу.
  • Визуальные индикаторы: Повышение цены выделяется зеленым цветом, а понижение — красным.
  • Интеграция с запасами (доступна при включённом модуле «Склад»): Просмотр текущих остатков (включая вложенные склады) для каждой позиции на выбранные даты и склады.
  • Фильтрация:
    • По Виду цены: Сосредоточьтесь на конкретном виде цены или просмотрите все.
    • По Месту хранения: Фильтруйте товары по их наличию в конкретном месте хранения.

Как использовать​

  1. Перейдите в раздел Продажи -> Отчетность -> Отчет по ценам.
  2. Выберите целевую Дату (дата и время).
  3. (Опционально) Выберите дату в поле Сравнить, чтобы увидеть изменения цен и запасов.
  4. (Опционально) Выберите Вид цены, чтобы отфильтровать отображаемые цены.
  5. (Опционально) Выберите Место хранения, чтобы увидеть остатки и отфильтровать товары, которые есть в наличии в этом месте хранения.
  6. Просмотрите результаты и при необходимости уточните параметры.

Отчет отображает список товаров с их атрибутами, штрих-кодами, единицами измерения, уровнями запасов и ценами на выбранные даты.

Параметры и интерпретация​

  • Дата: момент времени, на который берутся значения цен.
  • Сравнить: дополнительный момент времени. Если задан, отчет показывает цены для обеих дат и % разницы.
  • Вид цены: ограничивает вывод одним видом цены. Если не задан, отчет показывает все настроенные виды цен.
  • Место хранения: определяет:
    • какой остаток показывается;
    • какие товары считаются «в наличии» для фильтра. Остаток показывается с учетом вложенных мест хранения.

Изменения подсвечиваются:

  • повышение цены — зеленым;
  • снижение цены — красным.

Типовые сценарии​

  • Проверка изменения цен: задайте Дату до и Сравнить после изменения цен, чтобы подтвердить ожидаемую разницу.
  • Аудит истории цен: используйте разные значения Даты, чтобы понять, когда цена изменилась.
  • Планирование пополнения: выберите Место хранения, чтобы сфокусироваться на товарах в наличии (или с нулевым остатком) в выбранном месте хранения.

Советы​

  • Используйте сопоставимые моменты времени (например, конец дня vs конец дня), чтобы избегать некорректных сравнений.
  • При анализе остатков всегда задавайте Место хранения, чтобы колонка остатков была информативной.
  • Если у вас несколько видов цен, начните с фильтрации по Виду цены, чтобы не смешивать разные правила ценообразования.

Рекомендации​

  1. Начните с узких параметров (один Вид цены, одно Место хранения), затем расширяйте выборку при необходимости.
  2. Экспортируйте или копируйте результаты, если нужно поделиться выводами или сохранить след аудита.
  3. Если цифры выглядят неожиданно, проверьте, что выбрана нужная Дата (включая время) и корректное Место хранения.

Отчет по заказам​

Отчет по заказам дает детальный обзор заказов покупателей и их строк. Подходит для оперативного анализа (что заказано, кем, для какого места хранения/даты) и для экспорта данных в таблицы.

Как использовать​

  1. Перейдите в Продажи -> Отчетность -> Отчет по заказам.
  2. (Опционально) В разделе Фильтры задайте Интервал дат, чтобы ограничить заказы по дате заказа.
  3. Используйте фильтрацию/сортировку таблицы, чтобы сфокусироваться на покупателе, статусе, подразделении, месте хранения, номенклатуре и т.д.

Параметры и интерпретация​

  • Интервал дат: фильтрует заказы по Дате заказа (включительно). Если не задан, отчет показывает все доступные заказы.

Результат​

Отчет показывает информацию заголовка заказа и информацию по строкам (поля заголовка повторяются для каждой строки):

  • Заказ: номер, дата/время, покупатель, статус, тип, ссылка/референс покупателя, подразделение, адрес покупателя, адрес доставки, плановая дата/время, место хранения, условия оплаты, представитель и другие поля заголовка.
  • Строки: номенклатура, описание, количество, единица измерения, цена, скидка, цена со скидкой, налоги и суммы (без налога, налог, итого).
  • Себестоимость и маржа (при включённом учёте себестоимости отгрузок): сумма себестоимости, сумма наценки и наценка по строке.
  • Категории и атрибуты: уровни категорий и настроенные атрибуты номенклатуры (как отдельные колонки).

Типовые сценарии​

  • Оперативный экспорт: выгрузка списка строк заказов для склада/логистики.
  • Аудит заказов: сравнение скидок, налогов и итогов по покупателям или подразделениям.
  • Анализ ассортимента: сегментация по категориям и атрибутам.

Советы​

  • Начинайте с Интервала дат, чтобы отчет был быстрее и удобнее.
  • При большом количестве атрибутов отчет может быть очень широким — используйте горизонтальную прокрутку или экспорт.

Отчет по продажам​

Отчет по продажам — детальный отчет на основе записей регистра продаж. Он объединяет показатели продаж с себестоимостью и маржой и может использоваться для анализа прибыльности.

Как использовать​

  1. Перейдите в Продажи -> Отчетность -> Отчет по продажам.
  2. В разделе Фильтры задайте Интервал дат, чтобы ограничить период отчета.
  3. Используйте фильтрацию/сортировку, чтобы сфокусироваться на покупателе, месте хранения, номенклатуре, категории или номере документа.

Параметры и интерпретация​

  • Интервал дат: фильтрует записи регистра по дате (включительно).
  • Показатели (типовое значение):
    • Количество — проданное количество в единице измерения номенклатуры.
    • Сумма себестоимости — себестоимость проданных товаров.
    • Сумма маржи — маржа в абсолютном выражении.
    • Сумма без налога, Сумма налога, Сумма — выручка с разбиением по налогу.

Результат​

Отчет обычно включает:

  • Идентификацию документа: класс/тип, дата/время, номер.
  • Измерения: покупатель, место хранения, номенклатура, уровни категорий, атрибуты номенклатуры.
  • Цена/маржа: цена, наценка.
  • Показатели: количество, сумма себестоимости, сумма маржи, суммы без налога/налога/итого.

Дополнительные колонки и разделы (если включено)​

В зависимости от включенных функций отчет может включать:

  • Детали скидок: полная цена, скидка продажи, сумма скидки.
  • Вкладку Продажи по статьям: сводная таблица по статье продаж, показывающая выручку, себестоимость и маржу.

Типовые сценарии​

  • Анализ прибыльности: маржа по номенклатуре/категориям/местам хранения.
  • Оборот покупателей: выручка, налоги и итоги по покупателям.
  • Проверка аномалий: фильтрация по необычно низкой/высокой марже.

Советы​

  • Всегда начинайте с Интервала дат.
  • Для сравнения мест хранения или категорий удобно экспортировать отчет и строить дополнительные своды в таблице.

Сводка по продажам​

Сводка по продажам — дашборд с графиками и сводными таблицами, который дает обзор трендов продаж на высоком уровне.

Как использовать​

  1. Перейдите в Продажи -> Отчетность -> Сводка по продажам.
  2. В разделе Фильтры задайте Интервал дат.
  3. Переключайтесь между вкладками, чтобы анализировать продажи по неделям, покупателям, местам хранения и категориям.

Параметры и интерпретация​

  • Интервал дат: ограничивает данные регистра продаж, используемые для построения графиков.

Что вы увидите​

Дашборд содержит четыре вкладки:

  • Продажи по неделям: один график (линейный) с трендами оборота и маржи по неделям.
  • Продажи по покупателям: гистограмма с рейтингом продаж по покупателям (оборот и маржа).
  • Продажи по местам хранения: два графика, один под другим — Оборот и Маржа — по неделям в разрезе мест хранения.
  • Продажи по категориям: две круговые диаграммы — Оборот и Маржа — по категориям (второй уровень иерархии).

Все графики используют один и тот же фильтр «Интервал дат» и работают на основе регистра продаж (SalesLedger).

Типовые сценарии​

  • Управленческий обзор: мониторинг оборота и маржи во времени.
  • Анализ сезонности: сравнение недельных паттернов между периодами.
  • Анализ структуры: понимание, какие покупатели/категории дают основной вклад в продажи.
